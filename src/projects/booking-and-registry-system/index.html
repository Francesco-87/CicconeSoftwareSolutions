Booking & Registration System

Full-Stack Web Application

  • Type: Booking system
  • Tech: HTML, CSS, JavaScript
View Client Repo View Server Repo

Overview

I designed and implemented a full-stack booking and registration system that enables customers to manage meal bookings and dietary preferences, while providing operational staff with structured and aggregated data for planning and preparation.

The system replaces manual scheduling processes with a centralized, data-driven solution.

Images

Challenge

Companies in the food and service industry often handle meal registrations and dietary requirements manually
This leads to:

  • Incomplete allergy tracking
  • Scheduling conflicts
  • Inefficient communication between customers and kitchen staff
  • Food overproduction and waste

The challenge was to build a system that:

  • Stores structured customer data
  • Tracks dietary restrictions and preferences
  • Allows booking and cancellation of meals
  • Provides summarized operational data for daily preparation

The solution needed to be responsive, scalable, and maintain clear separation between frontend and backend logic.

Solution

Architecture

The application follows a classic full-stack architecture:

Frontend

  • HTML / CSS
  • Responsive UI
  • Dynamic client-side behavior

Backend

  • Node.js server
  • REST-based communication
  • Structured MySQL database

Deployment

  • Docker-based containerization

This separation ensures maintainability and clear responsibility between presentation, business logic, and persistence layers.

Database Design

The relational database was structured around core entities:

  • Customers
  • Organizers / Staff
  • Bookings
  • Allergens
  • Menus

The system supports:

  • Role-based access (customer vs. organizer)
  • CRUD operations across all main entities
  • Query-based aggregation for daily summaries

The data model allows both individual-level insight (single customer preferences) and aggregated operational insight (daily allergen totals).

Key Features

Customer

  • Account creation and management
  • Allergen and dietary preference selection
  • Meal booking and cancellation
  • Personal data updates

Organizer / Staff

  • View filtered booking lists
  • Access summarized allergen data
  • Manage menus
  • Retrieve structured operational overviews

Outcome

The result is a fully functioning full-stack application that:

  • Centralizes booking and dietary data
  • Reduces communication overhead
  • Improves preparation accuracy
  • Supports operational planning through aggregated insights

The system demonstrates:

  • Structured relational database modeling
  • Backend–frontend integration
  • Clean separation of concerns
  • Deployment-ready architecture

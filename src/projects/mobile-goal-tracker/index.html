Mobile Goal Tracker

Android Progression Tracker App

  • Industry: Fitness / Self-improvement
  • Type: Mobile application
  • Tech: Java, Android Studio, XML, Material Design, JSON, JUnit 5, Mockito
View Repository

Overview

I developed an Android application that lets users set personal goals and track workout progress by recording planned and completed workouts. The data is processed and saved so the user can review performance over time.

The UI follows Material Design and Android app quality guidelines, with multiple screens for planning, managing workouts, and viewing performance.

Challenge

Build a simple, easy-to-use tracking app that helps users define a goal, enter performance data, preserve it reliably, and display progress clearly across multiple screens.

  • Support goal definition and workout planning/execution tracking
  • Provide clear navigation across several Android activities
  • Persist data locally and restore it for display and calculations
  • Evaluate progress (e.g., current period vs previous periods) and visualize results
  • Include unit testing for core logic

Solution

Architecture

The app is structured around dedicated Android activities, with models for core data objects and utility classes for reusable logic. Navigation is handled via button click listeners and intents.

Frontend

  • Android UI built with XML layouts and Material Design components
  • Dashboard with navigation to Planning, Performance, and Manage Workouts screens
  • Planning view uses a calendar/date picker to show planned, missed, and fulfilled workouts
  • Performance view visualizes progress with progress bars and shows the last three workouts
  • Manage Workouts view supports adding workouts via dialog and deleting via long-press

Backend

  • On-device processing in Java (no external backend)
  • Progress calculations implemented via reusable utilities (sorting, filtering, date-based counting)

Deployment

  • Android app build and run workflow through Android Studio
  • Version-controlled with GitHub

This separation keeps UI code focused on views and events, while shared logic (dates, arrays, persistence) remains testable and maintainable.

Database Design

Instead of a remote database, the app preserves data in internal storage using JSON as a unified format. Data is written and read as strings and then converted back into object lists for rendering and calculations.

  • Models: WorkoutData (date, name) and GoalObject (start date, end date, target)
  • JSONConversion utility converts between JSON and object lists
  • FileManager utility stores, retrieves, checks, and deletes internal files

The stored data is reused for calendar display, progress visualizations, and “last workouts” summaries.

Key Features

User

  • Create and manage workout plans
  • Set a goal for a chosen timeframe
  • Track workouts on a calendar view
  • View performance summaries (Current, Last Week, Last Month)
  • See a list of recent workouts

App Logic

  • User input via dialogs/popups (workout name, goal values and dates)
  • Local persistence using JSON + internal storage
  • Progress processing via date-based list evaluation and counting
  • Unit testing for models and selected logic using JUnit 5 (with Mockito where needed)

Outcome

The result is a functional Android progression tracker that lets users define goals, record workout data, preserve it locally, and visualize progress through clear UI panels.

  • Goal setting and workout tracking across multiple activities
  • Calendar-based planning and status overview
  • Performance visualization with progress bars and recent activity summaries
  • Reliable persistence using JSON in internal storage
  • Unit tests validating key parts of the implementation

The system demonstrates:

  • Multi-activity Android app design
  • Separation of concerns (Activities, Models, Utilities)
  • Local persistence strategy with JSON conversion
  • Date-driven evaluation and progress calculation
  • Testing with JUnit 5 and Mockito

Technical Highlights

  • Material Design-based UI and navigation
  • Calendar/date picker planning flow
  • JSON serialization/deserialization for stored lists
  • Internal storage read/write utilities
  • Array utilities for sorting/filtering recent workouts
  • Unit testing strategy for Android-adjacent logic
